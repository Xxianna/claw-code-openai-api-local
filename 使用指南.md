# Claw Code 使用指南

Claw Code 是一个高性能的 Rust 实现的 CLI 代理工具，用于与 AI 模型（如 Anthropic 的 Claude、OpenAI 兼容的 Qwen 等）交互。

## 快速开始

### 1. 构建项目

```bash
cd rust
cargo build --workspace
```

构建完成后，CLI 二进制文件位于 `rust/target/debug/claw`

### 2. 首次运行健康检查

```bash
cd rust
./target/debug/claw
# 在 REPL 中输入
/doctor
```

## 身份验证

### 方式一：使用 API 密钥（Anthropic）

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
```

### 方式二：使用 API 密钥（OpenAI 兼容 API）

```bash
export OPENAI_API_KEY="your-api-key"
export OPENAI_BASE_URL="http://your-api-endpoint/v1"
```

### 方式三：使用 OAuth

```bash
cd rust
./target/debug/claw login
```

## 使用方式

### 交互式 REPL 模式

```bash
cd rust
./target/debug/claw
```

### 指定模型启动

```bash
# 使用 Anthropic 模型
./target/debug/claw --model claude-sonnet-4-6

# 使用 Qwen 模型（OpenAI 兼容）
./target/debug/claw --model Qwen/Qwen3.5-27B-FP8
```

### 一次性提示

```bash
./target/debug/claw --model Qwen/Qwen3.5-27B-FP8 prompt "你是谁"
```

### 快捷模式

```bash
./target/debug/claw --model Qwen/Qwen3.5-27B-FP8 "你是谁"
```

### JSON 输出（用于脚本）

```bash
./target/debug/claw --output-format json prompt "status"
```

## OpenAI 兼容 API 配置

项目已支持 OpenAI 兼容的 API（如 Qwen、vLLM 等），可以通过环境变量配置高级参数。

### 基本配置

```bash
export OPENAI_API_KEY="your-api-key"
export OPENAI_BASE_URL="http://192.168.0.195:8000/v1"
```

### 采样参数配置

```bash
# 温度参数（控制随机性，0-2 之间）
export CLAW_TEMPERATURE=0.7

# 核采样参数（0-1 之间）
export CLAW_TOP_P=0.8

# 存在惩罚（-2 到 2 之间）
export CLAW_PRESENCE_PENALTY=1.5

# 频率惩罚（-2 到 2 之间）
export CLAW_FREQUENCY_PENALTY=0.0

# Top-K 采样
export CLAW_TOP_K=20
```

### 额外参数（JSON 格式）

```bash
# 禁用思考模式
export CLAW_EXTRA_BODY='{"chat_template_kwargs": {"enable_thinking": false}}'

# 同时设置多个参数
export CLAW_EXTRA_BODY='{"top_k": 20, "chat_template_kwargs": {"enable_thinking": false}}'
```

### 完整示例

```bash
cd rust

# 设置环境变量
export OPENAI_API_KEY="your-api-key"
export OPENAI_BASE_URL="http://192.168.0.195:8000/v1"
export CLAW_TEMPERATURE=0.7
export CLAW_TOP_P=0.8
export CLAW_EXTRA_BODY='{"chat_template_kwargs": {"enable_thinking": false}}'

# 启动
./target/debug/claw --model Qwen/Qwen3.5-27B-FP8
```

## 环境变量

### Anthropic 相关

| 变量名 | 说明 |
|--------|------|
| `ANTHROPIC_API_KEY` | Anthropic API 密钥 |
| `ANTHROPIC_BASE_URL` | Anthropic API 基础 URL（可选） |
| `ANTHROPIC_AUTH_TOKEN` | Anthropic OAuth 令牌（可选） |

### OpenAI 兼容 API 相关

| 变量名 | 说明 |
|--------|------|
| `OPENAI_API_KEY` | OpenAI 兼容 API 密钥 |
| `OPENAI_BASE_URL` | OpenAI 兼容 API 基础 URL |
| `XAI_API_KEY` | xAI API 密钥（用于 Grok 模型） |
| `XAI_BASE_URL` | xAI API 基础 URL（可选） |

### 采样参数

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `CLAW_TEMPERATURE` | 温度参数 | 无 |
| `CLAW_TOP_P` | 核采样参数 | 无 |
| `CLAW_PRESENCE_PENALTY` | 存在惩罚 | 无 |
| `CLAW_FREQUENCY_PENALTY` | 频率惩罚 | 无 |
| `CLAW_TOP_K` | Top-K 采样 | 无 |
| `CLAW_EXTRA_BODY` | 额外 JSON 参数 | 无 |

### 其他

| 变量名 | 说明 |
|--------|------|
| `CLAW_CONFIG_HOME` | 配置文件目录 |

## 会话管理

REPL 的对话会持久化保存在当前工作区的 `.claw/sessions/` 目录下。

### 恢复会话

```bash
# 恢复最新会话
./target/debug/claw --resume latest

# 恢复会话并执行命令
./target/debug/claw --resume latest /status /diff
```

## 配置文件

运行时配置按以下顺序加载，后面的覆盖前面的：

1. `~/.claw.json`
2. `~/.config/claw/settings.json`
3. `<repo>/.claw.json`
4. `<repo>/.claw/settings.json`
5. `<repo>/.claw/settings.local.json`

## 权限模式

- `read-only` - 只读模式
- `workspace-write` - 工作区写入模式

```bash
./target/debug/claw --permission-mode workspace-write prompt "update README.md"
```

## 常见问题

### Q: 如何使用 Qwen 模型？

A: 设置 `OPENAI_API_KEY` 和 `OPENAI_BASE_URL` 环境变量，然后使用 `--model Qwen/Qwen3.5-27B-FP8` 启动。

### Q: 如何禁用思考模式？

A: 设置 `CLAW_EXTRA_BODY='{"chat_template_kwargs": {"enable_thinking": false}}'` 环境变量。

### Q: 如何调整温度参数？

A: 设置 `CLAW_TEMPERATURE=0.7` 环境变量。
